# 疑难解答

## “不支持的浏览器”

我们推荐使用**最新的** Google Chrome 或 Microsoft Edge 浏览器。您也可以自行试验其他您常用的浏览器。

如果您在意隐私或广告，可以选择 Cromite 或 Vivaldi。

注意，受 Apple 限制，iOS 上的 Chrome 仍无法使用蓝牙功能。  
您需要前往 App Store 下载 [Bluefy](https://apps.apple.com/us/app/id1492822055) 浏览器以使用本项目。

## “未授予蓝牙权限。”

### 桌面（Windows/Linux/macOS）

请按顺序检查以下项目：

- 电脑已安装蓝牙硬件。
- 蓝牙驱动工作正常。
- 已在系统设置中开启蓝牙。
- 已在浏览器中为本站授权蓝牙权限。

### Android

请在手机设置中，为浏览器授予蓝牙权限。

在 Android 系统中，该权限被称作“附近设备”权限。

> [!NOTE]
> 注意：在 Android 11 及更低版本中，访问蓝牙设备的权限被归类在“位置信息”权限当中。因此，在旧版本 Android 我们可能需要您授予这一权限。

默认情况下，浏览器会提示用户授权附近设备（或位置信息）权限。但在一部分定制的 Android 系统中，这一权限默认是“否”，需要用户手动到系统设置中授权。

我们承诺不会利用这一权限对您进行跟踪或定位。

## 点击「开启」后没有设备 / 选不到水表

请按顺序检查：

- 手机 / 电脑蓝牙已打开，并尽量靠近水表（建议 1 米内）。
- 水表已上电、未被其他手机长期占用连接。
- 系统弹窗里选择的是**对应水表**，不要选成耳机、手环等其他 BLE 设备。
- 高级面板里的「设备名关键字」不要填得过严；默认可不填，交给系统选择器。
- 若刚在别的页面连过同一台设备，先点高级面板断开 / 清除已保存设备后再试。

## 连接成功但不开闸 / 显示开闸失败

常见原因：

- 当前未勾选「免费用水模式」。本页默认走小程序 `isFreeWater` 本地开闸，不走云余额。
- 连错设备，或写特征 / 通知特征 UUID 与该水表不匹配。
- 水表回了拒绝帧：展开高级面板，查看「最后指令 / 最后回包」与调试日志。
- 开闸命令不是默认 `85 / 00`：可在高级面板切换开闸命令后重试。
- 连接后 `23 / 00` 同步失败不一定致命，页面会继续尝试开闸；若持续失败，请导出日志排查。

## 已开启但不出水

- 开闸成功后请**立刻打开水龙头**；部分水表有短超时。
- 确认点的是本页「开启」，且状态显示「用水中 / 已开启」。
- 如果开闸后立刻点了「关闭」，会先发 `86 / 00` 结束用水，再发 `87 / 00` 关阀。
- 建议保持「开闸后保持连接（推荐）」勾选，避免过早断开影响持续出水。

## 关闭失败 / 关不掉

关闭流程对齐小程序：

1. 若当前正在用水：先发 `86 / 00`（结束用水）
2. 再发 `87 / 00`（仅关闭）
3. 若当前本来就没在用水：直接 `87 / 00`

若关闭失败：

- 确认设备仍保持连接；断开后可再点「关闭」触发重连。
- 查看调试日志中的结束回包 / 仅关闭回包。
- 靠近设备后重试，避免 BLE 链路中途断开。

## 页面提示需要 HTTPS

Web Bluetooth 只能在安全上下文使用：

- 推荐通过 GitHub Pages：https://bangsheng-hybb.github.io/bleworker/
- 或本地：`http://localhost:端口`
- `file://` 直接打开通常**没有**蓝牙权限

## 如何导出日志给开发者

1. 打开页面底部「高级」
2. 复现一次失败操作
3. 点击 **导出日志**（或复制页面中的定位日志整段）
4. 日志中会包含错误码、步骤、TX/RX、特征信息与最近步骤，便于直接定位

## 这和 waterctl 有什么关系？

[celesWuff/waterctl](https://github.com/celesWuff/waterctl) 是更完整的蓝牙水控器 FOSS 实现，文档与交互很优秀。  
`bleworker` 侧重单页、免费用水开闸链路与小程序协议对齐，README / FAQ 的组织方式参考了 waterctl，便于用户快速上手与排障。
